# AGENTS.md

Guidance for AI coding agents working in this repository.

## Repo overview

This is the official Terraform provider for ConductorOne. Most of the Go code is generated by Speakeasy from an OpenAPI spec — see `CLAUDE.md` for the full maintainer workflow (regen, validation, release).

## Skills

Specialized workflows live under `skills/`. Load the relevant skill when its trigger applies.

| Skill | When to use |
|---|---|
| [`skills/upgrading/SKILL.md`](skills/upgrading/SKILL.md) | A user needs to upgrade the `ConductorOne/conductorone` provider version in their Terraform / OpenTofu configuration. Covers reading the breaking-changes log, applying attribute migrations, validating with `terraform plan`, and recovering from known issues (notably the v1.1.0 bad release and the v1.2.0 `deleted_at` crash). |
| [`skills/release/SKILL.md`](skills/release/SKILL.md) | Cutting a new spec-driven release of the provider. Documents the regen sequence and the seven hand-maintained edges Speakeasy will not handle: `provider.go` registration, doc-emission ordering, README install-snippet version lag, pagination-fix verification, breaking-changes log in `templates/index.md.tmpl`, patch-tripwire file-list extension for newly-emitted resources that nest entity-root types (IGA-1774 class), and the human-tagged GitHub release plus post-tag note polish. Trigger on `/release`, "cut a release", "regen for vX.Y.Z", or any time a Speakeasy regen is about to ship. |

## Other key docs

- `CLAUDE.md` — repo architecture, codegen flow, Speakeasy version pin policy, and the maintainer release workflow.
- `KNOWN_ISSUES.md` — active upstream regressions affecting regeneration.
- `RELEASES.md` — release-cutting checklist.
- `USAGE.md` — end-user provider usage examples.
- `docs/index.md` — rendered provider docs, including the canonical **Breaking changes** section (source: `templates/index.md.tmpl`).

## Conventions

- Never hand-edit files with `// Code generated by Speakeasy` headers.
- After any `make gen` / `speakeasy run`, verify the pagination fix (see `CLAUDE.md`).
- Update `templates/index.md.tmpl` (not `docs/index.md` directly) and run `make generate` to refresh rendered docs.
- Run `make pre-commit` before pushing.
